# 阶段0：复现规格与中文日志规范

## 1. 目标边界

- 本阶段只建设仓库骨架与最小可运行链路，不进入 Stage1 功能开发。
- 以 `SPEC.md` 为规格基线，确保后续阶段可追溯、可复核、可返工。

## 2. 与 SPEC 对齐方式

- 图结构输出格式预留：后续统一为 `(n, [(u,v), ...])`。
- 分子输出与 RDKit 校验：仅在 Stage4 实装，本阶段只保留工程入口。
- 指标口径（valid/unique/novel、p_pred/p_gen、CM/C）在 Stage0 不计算，只在文档中固定术语。

## 3. 中文日志规范

- 面向用户文件（报告、日志、总结、报错说明）统一中文。
- 每次运行必须写入 `runs/<timestamp>/run.log`，并记录：
  - 运行时间
  - 执行命令
  - 成功/失败状态
  - 关键产物路径
- 阶段验收文件固定为 `runs/stageX/验收报告.md`，并附 `runs/stageX/关键输出/`。

## 4. 最小复核命令

```bash
python -m llm4graphgen.smoke
python -m pytest -q
make check
make stage0_summary
```
