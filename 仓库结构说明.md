### 2.2 各阶段主程序

#### `smoke.py`（Stage0，87 行）

**功能**：仓库骨架验证的最小 CLI 工具。

- `run_smoke()`：返回 "ok"，用于健康检查。
- `_render_tree()`：渲染目录树（深度 3），排除 `.git`/`__pycache__` 等。
- `build_summary()`：生成 Stage0 结构摘要，包含目录树和关键文件存在性检查。
- `main()`：CLI 入口，支持 `--summary` 参数。

#### `stage1_smoke.py`（Stage1，114 行）

**功能**：最小闭环验证，测试 Provider 抽象 + 图解析器 + JSONL 日志。

- `run_stage1()`：核心流程——选择 Provider（mock/openai）→ 调用 LLM → 解析输出 → 写入三个文件：
  - `llm_io.jsonl`：完整 LLM I/O 记录
  - `parser_result.json`：解析后的图结构
  - `run.log`：中文运行日志
- 支持通过 `--provider` 参数切换 Mock/OpenAI。

#### `stage2_rule_based.py`（Stage2，559 行）

**功能**：Rule-based 8 种图约束任务的评测引擎。

**核心数据结构**：
- `Graph`：不可变图结构（节点数 + 边元组）
- `TaskConfig`：任务配置（ID、名称、prompt、样本、训练集签名、验证器）

**8 个图验证器**：
| 函数 | 验证规则 |
|------|---------|
| `is_tree()` | 连通且边数 = n-1 |
| `is_cycle()` | 连通、所有度为 2、边数 = n |
| `is_planar_approx()` | 边数上界 + K₅/K₃,₃ 子图检查（近似） |
| `is_bipartite()` | BFS 二染色 |
| `is_k_regular()` | 所有节点度 = k |
| `is_wheel()` | 一个中心连接所有 + 外圈为 cycle |
| `is_k_colorable()` | 回溯着色算法 |
| `components_count()` | DFS 连通分量计数 |

**辅助函数**：
- `canonical_signature()`：生成边排序后的标准化图签名，用于 unique/novel 判定
- `adjacency()`：邻接表构建
- `build_graph()`：从 ParseResult 构建 Graph
- `validate_graph()`：根据 TaskConfig 分派到对应验证器
- `_task_configs()`：返回 8 个任务的完整配置（含 Mock 样本和 train_seen_signatures）

**输出**：`rule_based_metrics.csv`、`rule_based_samples.csv`、`failure_cases.csv`、`llm_io.jsonl`、`run.log`

#### `stage3_distribution.py`（Stage3，317 行）

**功能**：Distribution-based 3 种分布任务的评测引擎。

**核心数据结构**：
- `DistributionTaskConfig`：任务配置（ID、名称、prompt、样本、判别器）

**3 个分布判别器**：
| 函数 | 判别规则 |
|------|---------|
| `judge_trees_or_cycles()` | 图是 tree 或 cycle |
| `judge_union_of_components()` | 至少 2 个连通分量，每个为 tree 或 cycle |
| `judge_motif()` | 包含三角形（O(n³) 遍历检查），n>40 时拒绝 |

**辅助函数**：
- `_component_nodes()`：DFS 提取各连通分量节点集合
- `_induced_subgraph()`：提取诱导子图并重新编号
- `_canonical_edges()`：边标准化

**指标计算**：
- `p_pred = pred_positive / judge_success`
- `p_gen = pred_positive / total`
- `parse_fail_rate`、`judge_fail_rate`

**输出**：`distribution_metrics.csv`、`distribution_samples.csv`、`failure_cases.csv`、`llm_io.jsonl`、`run.log`

#### `stage4_property.py`（Stage4，261 行）

**功能**：Property-based MolHIV 分子生成评测引擎。

**核心数据结构**：
- `SmilesParseResult`：SMILES 解析结果（成功标志、规范化 SMILES、失败原因）

**关键函数**：
| 函数 | 功能 |
|------|------|
| `parse_smiles()` | RDKit SMILES 解析 + 规范化 |
| `_fingerprint_array()` | Morgan 指纹生成（radius=2, size=1024） |
| `_few_shot_dataset()` | 返回 11 个 few-shot 分子（5 正 + 6 负） |
| `train_baseline_classifier()` | Logistic Regression 训练 |
| `_generated_mock_outputs()` | 返回 12 个 Mock 分子输出 |

**指标计算**：
- `valid_rate`、`unique_rate`、`novel_rate`
- `C = mean(valid_labels)`（正类预测比例）
- `CM = mean(valid_scores)`（正类概率均值）
- `rectified_C = C × valid_rate`

**依赖**：`rdkit`、`numpy`、`scikit-learn`

**输出**：`property_metrics.csv`、`property_samples.csv`、`failure_cases.csv`、`baseline_train_summary.csv`、`llm_io.jsonl`、`run.log`

### 2.3 解析器模块 `parsers/`

| 文件 | 作用 |
|------|------|
| `__init__.py` | 导出 `GraphParseResult`、`parse_graph_output` |
| `graph_parser.py`（71 行） | 图结构输出解析器，解析 `(n, [(u,v), ...])` 格式。使用 `ast.literal_eval()` 做安全解析，支持容错（空格/换行/尾逗号/重复边/边反转）。所有错误信息为中文。返回 `GraphParseResult`（成功标志、节点数、规范化边列表、失败原因） |

### 2.4 Provider 模块 `providers/`

| 文件 | 作用 |
|------|------|
| `__init__.py` | 导出 `BaseProvider`、`MockProvider`、`OpenAIProvider` |
| `base.py`（17 行） | Provider 抽象基类，定义 `name` 属性和 `generate(prompt, model, temperature) -> str` 抽象方法 |
| `mock_provider.py`（22 行） | Mock 实现，返回固定输出字符串，忽略所有参数。默认输出 `(4, [(0,1), (1,2), (2,3), (0,1),])`。用于无 API 环境下的测试 |
| `openai_provider.py`（84 行） | OpenAI API 实现，从环境变量 `OPENAI_API_KEY` 读取密钥，使用 Responses API（POST `https://api.openai.com/v1/responses`），支持错误处理（HTTP/网络/解析错误均返回中文提示） |

## 七、数据流与执行流程

```
用户执行 python -m llm4graphgen.stageX
         │
         ▼
    ┌─────────────┐
    │  Provider    │  ← MockProvider（固定输出）或 OpenAIProvider（真实 LLM）
    │  .generate() │
    └──────┬──────┘
           │ raw_output (字符串)
           ▼
    ┌─────────────┐
    │  Parser     │  ← graph_parser.py (Stage2/3) 或 parse_smiles (Stage4)
    │  .parse()   │
    └──────┬──────┘
           │ GraphParseResult / SmilesParseResult
           ▼
    ┌─────────────┐
    │  Validator   │  ← is_tree/is_cycle/... (Stage2) 或 judge_xxx (Stage3) 或 classifier (Stage4)
    │  /Judge/Clf  │
    └──────┬──────┘
           │ is_valid / judge_positive / classifier_score
           ▼
    ┌─────────────┐
    │  Metrics    │  ← valid_rate / unique_rate / novel_rate / p_pred / p_gen / C / CM
    │  计算 + 导出 │
    └──────┬──────┘
           │
           ▼
    runs/<timestamp>/
    ├── llm_io.jsonl
    ├── metrics.csv
    ├── samples.csv
    ├── failure_cases.csv
    └── run.log
```
