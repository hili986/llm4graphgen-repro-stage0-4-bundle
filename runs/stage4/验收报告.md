# Stage4 验收报告

## 1) 阶段目标（一句话）

完成 Property-based（MolHIV）最小可复现流程：RDKit 合法性校验、Unique/Novel、baseline 分类器、CM/C/rectified C 统计与验收出表。

## 2) 完成清单（逐条）

- 已安装并使用 RDKit（`rdkit-2025.9.4`）执行 SMILES 合法性校验。
- 已实现 Stage4 主程序：`src/llm4graphgen/stage4_property.py`。
- 已实现 baseline 分类器（few-shot 训练集 + Morgan 指纹 + LogisticRegression）。
- 已实现指标计算：
  - `Unique = unique_valid_count / valid_count`
  - `Novel = novel_unique_count / unique_valid_count`
  - `C = 有效样本中 baseline 判为正类比例`
  - `CM = 有效样本中 baseline 正类概率均值`
  - `rectified_C = C * valid_rate`
- 已实现样本级导出：`property_samples.csv`。
- 已实现汇总导出：`property_metrics.csv`。
- 已实现失败样例导出：`failure_cases.csv`。
- 已实现 LLM I/O 落盘：`llm_io.jsonl`（含 prompt/model/temperature/raw_output/parsed_result/parse_failure_reason）。
- 已新增 Stage4 单测：`tests/test_stage4_property.py`。

## 5) 偏差与不确定性（明确说明原因）

- MolHIV few-shot 训练数据在本阶段使用最小代理样本集，不是 OGBG-MolHIV 官方划分。
- baseline 分类器为复现近似方案（Morgan 指纹 + 逻辑回归），不代表论文中可能使用的完整分类器细节。
- Novel 判定口径基于本地 few-shot 训练集，而非完整训练语料，数值仅用于流程复现。
- 当前阶段以 Mock 生成输出驱动评测链路，结果不用于对齐论文绝对数值。
- RDKit 对非法 SMILES 会输出底层解析警告（英文 stderr），不影响 CSV/JSONL 的中文失败原因记录。
