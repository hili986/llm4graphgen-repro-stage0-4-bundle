## 一、核心复现代码与论文原文的符合度分析

### 1.1 总体架构对比

论文提出了一个系统性的 LLM 图生成评估框架，包含三大评测维度：

| 维度 | 论文设计 | 复现实现 | 符合度 |
|------|---------|---------|--------|
| Rule-based | 8 种约束任务，4 种 prompting 策略 | 8 种约束任务，仅 Mock 数据 | 部分符合 |
| Distribution-based | 3 种分布任务，p 值范围 0.2-0.8 | 3 种分布任务，固定样本 | 部分符合 |
| Property-based | MolHIV + GNN 分类器 | MolHIV 近似 + LR 分类器 | 框架符合，细节偏差 |

### 1.2 Rule-based（Stage2）代码分析

#### 符合论文之处

1. **8 种图约束任务完整实现**：Tree、Cycle、Planar、#Components、k-regular、Wheel、Bipartite、k-coloring 全部实现，与论文 Table 7 的 8 种任务一一对应。
2. **评估指标正确**：`valid_rate`、`unique_rate`、`novel_rate` 三个核心指标的计算口径与论文一致：
   - `valid_rate = valid_count / total_samples`
   - `unique_rate = unique_valid / valid_count`
   - `novel_rate = novel_unique / unique_valid`
3. **图输出格式**：采用 `(n, [(u,v), ...])` 边列表格式，与论文描述的 edge list 表示一致。
4. **图验证算法基本正确**：
   - Tree：连通 + 边数 = n-1（正确）
   - Cycle：连通 + 所有节点度为 2 + 边数 = n（正确）
   - Bipartite：BFS 二染色（正确）
   - k-regular：所有节点度检查（正确）
   - Wheel：中心节点度 = n-1 + 外圈为 cycle（正确）
   - k-coloring：回溯着色算法（正确，但效率可优化）

#### 与论文不符之处

1. **图规模严重不匹配**：
   - 论文 Table 7 明确规定：Trees/Cycles/Planar/Wheel 为 **15 节点**，k-regular 为 **16 节点**（k=3），Bipartite 为 **每个分区 5 节点（共 10 节点）**，k-coloring 为 **15 节点 32 边**。
   - 复现代码中 Mock 样本仅使用 **4-6 节点** 的小图，远小于论文设定。

2. **样本量极度不足**：
   - 论文每个任务生成 **100 个图**，并进行多次重复实验取均值和标准差。
   - 复现代码每任务仅 **6 个** Mock 样本，且无重复实验。

3. **缺少 Prompting 策略变体**：
   - 论文测试了 **4 种策略**：Zero-shot、Few-shot、Zero-shot+CoT、Few-shot+CoT。
   - 复现代码仅使用单一 Mock 输出，未实现任何 prompting 策略区分。

4. **Planar 图判定为近似算法**：
   - 复现使用 `is_planar_approx()`，仅检查 K₅/K₃,₃ 子图 + 边数上界，这是 **Kuratowski 定理的必要条件检查（非充分条件）**。
   - 论文隐含使用精确平面性检测算法（如 Hopcroft-Tarjan 或 Boyer-Myrvold）。
   - K₅/K₃,₃ 子图检查的暴力枚举复杂度为 O(n⁵) 和 O(n⁶)，对论文要求的 15 节点图（15 选 5 = 3003，15 选 6 = 5005）尚可接受，但检查的是子图而非 subdivision，这在理论上不完全等价于平面性判定。

5. **缺少图规模消融实验**：
   - 论文 Table 4/8 测试了 Small/Medium/Large 三种规模。
   - 复现代码完全没有实现这一维度的实验。

6. **Novel 判定基线不对齐**：
   - 论文的 "novel" 基于 few-shot 提供的示例图 + 训练集中可见的图。
   - 复现代码使用手动构造的 `train_seen_signatures`，每个任务仅 1-2 个签名。

### 1.3 Distribution-based（Stage3）代码分析

#### 符合论文之处

1. **三种子任务完整实现**：Trees-or-cycles、Union-of-components、Motif（triangle）。
2. **Judge 函数逻辑正确**：
   - `judge_trees_or_cycles`：判断图是 tree 或 cycle（正确）。
   - `judge_union_of_components`：每个连通分量为 tree 或 cycle（正确）。
   - `judge_motif`：三角形检测（正确）。
3. **失败率追踪完整**：`parse_fail_rate` 和 `judge_fail_rate` 均有统计，符合 SPEC 要求。

#### 与论文不符之处

1. **缺少 p 值扫描实验**：
   - 论文核心实验是在 **p = 0.2, 0.4, 0.6, 0.8** 四个分布参数下测试，观察 LLM 对分布的推断能力。
   - 复现代码完全没有 p 值参数化设计，仅用固定 Mock 样本。

2. **实验设计与论文不同**：
   - 论文流程：给 LLM **10 个输入图**（按某分布采样），要求 LLM **推断 p 值并生成 10 个新图**。
   - 复现代码：直接对固定 Mock 输出做 judge，没有"输入图→推断 p→生成"的完整流程。

3. **Motif 任务简化**：
   - 论文使用多种 base 图（Trees/Ladders/Wheels）+ 多种 motif（Cycles/Houses/Cranes）的组合。
   - 复现仅实现 triangle 一种 motif。

4. **指标口径差异**：
   - 论文的 `p_pred` 是 LLM 推断出的分布参数 p 值，`p_gen` 是从 LLM 生成图中计算出的经验分布参数。
   - 复现的 `p_pred` 实际是 `pred_positive_count / judge_success_count`（正类率），`p_gen` 是 `pred_positive_count / total_samples`（生成正类率），含义不完全对等。

### 1.4 Property-based（Stage4）代码分析

#### 符合论文之处

1. **SMILES 解析与 RDKit 校验**：使用 `Chem.MolFromSmiles()` + `Chem.MolToSmiles()` 进行合法性检查和规范化，与论文一致。
2. **核心指标实现**：
   - `C`（正类预测比例）、`CM`（正类概率均值）、`rectified_C = C × valid_rate` 的计算与论文定义一致。
   - `Unique` 和 `Novel` 率计算正确。
3. **Morgan 指纹**：radius=2, fp_size=1024 是分子指纹的标准配置。

#### 与论文不符之处

1. **分类器差异**：
   - 论文使用 **GNN（图神经网络）** 在完整 OGBG-MolHIV 数据集（~41,127 样本）上训练的分类器。
   - 复现使用 **Logistic Regression + Morgan Fingerprints**，仅在 **11 个 few-shot 样本**上训练。
   - 这是最关键的偏差：分类器性能直接影响 C/CM/rectified_C 指标。

2. **数据集差距**：
   - 论文使用 OGBG-MolHIV 官方划分的训练集和测试集。
   - 复现使用 5 个正类 + 6 个负类的 **手动构造 few-shot 集**，不代表真实数据分布。

3. **rectified_C 计算可能偏差**：
   - 论文的 rectified C 考虑了分类器本身的误差率（Table 10 混淆矩阵），即 `C(G) = (C_M(G) - FPR) / (TPR - FPR)`。
   - 复现代码的 `rectified_C = C * valid_rate` 是一个简化近似，没有使用分类器的 FPR/TPR 校正。

4. **Prompt 设计差异**：
   - 论文使用 Few-shot 和 Few-shot+CoT 两种策略，给出真实 MolHIV 正类分子作为示例。
   - 复现使用固定 Mock 输出，没有真实 LLM 交互。

### 1.5 通用层面的差距

| 方面 | 论文 | 复现 | 差距程度 |
|------|------|------|---------|
| LLM 模型 | GPT-4, GPT-3.5, LLama2-13B | MockProvider（固定输出） | **极大** |
| 采样温度 | t=0.5-0.8，有消融实验 | 固定 t=0.0 | **大** |
| 样本量 | 100/task（多次重复） | 6-12/task（单次） | **极大** |
| Prompting | 4 种策略对比 | 无 | **极大** |
| 统计显著性 | 均值±标准差 | 单次运行 | **大** |

   - 复现的 valid_rate 集中在 0.50-0.67，而论文在 13%-100% 之间跨度极大。

3. **unique_rate 偏低**：
   - 复现约 0.67，论文大多在 85%-100%。
   - 原因：Mock 样本中有故意重复的样本，导致 unique 率被压低。

4. **novel_rate 无法有效对比**：
   - 复现的 train_seen_signatures 仅有 1-2 个签名，基线不对齐。

   - 论文使用分类器混淆矩阵校正：`C(G) = (C_M(G) - FPR) / (TPR - FPR)`。
   - 复现使用简化公式：`rectified_C = C × valid_rate`。
   - 这两个公式语义完全不同，数值不可对比。

## 三、进一步提升复现效果的可执行方法

### 3.1 优先级一：接入真实 LLM（收益最大）

**目标**：将 Mock 输出替换为真实 LLM 生成。

**具体步骤**：

1. **设置 OpenAI API Key**：
   ```bash
   export OPENAI_API_KEY="sk-..."
   ```

2. **修改各 Stage 的执行逻辑**，使其调用真实 OpenAIProvider：
   - 在 `stage2_rule_based.py` 中，将 `task.samples` 替换为对 LLM 的实时调用。
   - 在 `run_stage2()` 函数中添加 `provider` 参数，循环调用 `provider.generate(prompt, model, temperature)` 生成 100 个样本。

3. **实现 4 种 Prompting 策略**：
   - Zero-shot：直接提问。
   - Few-shot：在 prompt 前附加 2-3 个示例图。
   - Zero-shot+CoT：添加 "Let's think step by step" 引导。
   - Few-shot+CoT：示例 + 推理引导组合。

4. **对齐采样温度**：Rule-based 用 `t=0.8`，Distribution/Property 用 `t=0.5`。

5. **代码改动示例**（`stage2_rule_based.py`）：
   ```python
   def run_stage2_live(provider, output_root, run_id, model, temperature,
                       num_samples=100, num_repeats=3):
       for repeat in range(num_repeats):
           for task in tasks:
               for i in range(num_samples):
                   raw_output = provider.generate(task.prompt, model, temperature)
                   # ... 解析和评估逻辑不变
   ```

### 3.2 优先级二：对齐图规模参数

**目标**：使任务配置与论文 Table 7 完全一致。

**具体步骤**：

1. **修改 `_task_configs()` 中的 prompt**，指定正确的图规模：
   - Tree: "Generate a tree with **15** nodes"
   - Cycle: "Generate a cycle with **15** nodes"
   - k-regular: "Generate a **3-regular** graph with **16** nodes"
   - Bipartite: "Generate a bipartite graph with **5** nodes in each partition"
   - k-coloring: "Generate a **3-colorable** graph with **15** nodes and **32** edges"

2. **添加图规模消融实验**（对标 Table 4/8）：
   ```python
   SIZE_CONFIGS = {
       "small":  {"cycles": 10, "k_regular": 12, "k_coloring": (10, 20)},
       "medium": {"cycles": 15, "k_regular": 16, "k_coloring": (15, 32)},
       "large":  {"cycles": 20, "k_regular": 20, "k_coloring": (18, 39)},
   }
   ```

### 3.3 优先级三：替换 Stage4 分类器

**目标**：使用论文级别的 GNN 分类器，或至少使用更大规模的训练数据。

**具体步骤**：

1. **安装 OGB 数据集**：
   ```bash
   pip install ogb
   ```

2. **替换 `_few_shot_dataset()`**，加载 OGBG-MolHIV 真实划分：
   ```python
   from ogb.graphproppred import GraphPropPredDataset
   dataset = GraphPropPredDataset(name="ogbg-molhiv")
   split = dataset.get_idx_split()
   train_idx = split["train"]
   ```

3. **训练更好的分类器**（可选方案）：
   - 方案 A：仍用 LR + Morgan Fingerprints，但在完整训练集上训练。
   - 方案 B：使用 GIN（Graph Isomorphism Network）或 GCN 训练，对齐论文的 GNN 方案。
   - 方案 C：直接下载 OGB 官方预训练的 GNN checkpoint。

4. **修正 rectified_C 公式**：
   ```python
   # 论文的 rectified C 定义
   fpr = fp / (fp + tn)  # 从混淆矩阵计算
   tpr = tp / (tp + fn)
   rectified_c = (cm - fpr) / (tpr - fpr)
   ```

### 3.4 优先级四：完善 Distribution-based 实验

**目标**：实现完整的参数化分布实验。

**具体步骤**：

1. **实现图分布采样器**：
   ```python
   def sample_trees_or_cycles(n, p_tree, num_samples=10):
       """按概率 p 生成 tree 或 cycle 的混合样本。"""
       graphs = []
       for _ in range(num_samples):
           if random.random() < p_tree:
               graphs.append(random_tree(n))
           else:
               graphs.append(random_cycle(n))
       return graphs
   ```

2. **实现完整实验流程**：
   ```
   for p in [0.2, 0.4, 0.6, 0.8]:
       input_graphs = sample_from_distribution(p)
       prompt = f"Given these 10 graphs: {format(input_graphs)}\n
                  Infer the distribution parameter and generate 10 new graphs."
       output = provider.generate(prompt, model, temperature)
       p_pred = extract_predicted_p(output)
       p_gen = calculate_empirical_p(parsed_graphs)
   ```

3. **扩展 Motif 任务**：
   - 添加 House motif（5 节点，6 边）和 Crane motif（5 节点，7 边）。
   - 添加 Ladder、Wheel 等 base 图类型。

### 3.5 优先级五：实现精确平面性检测

**目标**：替换 `is_planar_approx()` 为精确算法。

**具体步骤**：

1. **安装 NetworkX**（已有成熟实现）：
   ```bash
   pip install networkx
   ```

2. **替换平面性检测函数**：
   ```python
   import networkx as nx

   def is_planar(graph: Graph) -> bool:
       G = nx.Graph()
       G.add_nodes_from(range(graph.n))
       G.add_edges_from(graph.edges)
       return nx.check_planarity(G)[0]
   ```

3. NetworkX 使用 **Boyer-Myrvold** 算法，时间复杂度 O(n)，远优于当前的 O(n⁶) 暴力枚举。

### 3.6 优先级六：添加统计显著性

**目标**：使结果可与论文的 "均值±标准差" 格式对比。

**具体步骤**：

1. **多次重复实验**：每种配置至少运行 3-5 次。
2. **计算均值和标准差**：
   ```python
   import numpy as np
   results = [run_experiment(seed=i) for i in range(5)]
   mean = np.mean(results)
   std = np.std(results)
   print(f"{mean:.1f}±{std:.1f}")
   ```
3. **记录随机种子**：确保结果可复现。

### 3.7 改进路线图（推荐执行顺序）

```

  ↓
Step 2: 接入真实 LLM + 实现 4 种 Prompting 策略（需 API Key）
  ↓
Step 3: 扩大样本量至 100/task + 多次重复实验
```
